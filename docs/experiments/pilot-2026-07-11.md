## Variant matrix attempt #1 ‚Äî invalidated, not a real result

All 10 variant runs (5 hypotheses √ó 2 reps) failed within 3-5s at `exit=1`,
`total_cost_usd=0`, `num_turns=1` ‚Äî **not** a finding about any hypothesis.
Root cause, confirmed via `.eval-stderr.log`/`.eval-result.json`: the
account hit a session-level rate limit (`api_error_status: 429`,
`"You've hit your session limit"`) immediately after the baseline reps
consumed the available budget, so every subsequent `claude -p` call failed
before making any real API request. A second, independent bug was also
found and fixed regardless: each new scratch fixture dir is an untrusted
Claude Code workspace, so `settings.json`'s `permissions.allow` was being
silently ignored ("this workspace has not been trusted") ‚Äî `scaffold.sh`
now pre-trusts each scratch dir via `~/.claude.json`
(`projects.<path>.hasTrustDialogAccepted = true`), and
`eval/harness/cleanup.sh` removes that entry again when a scratch dir is
torn down. These 10 invalid rows were not left in `eval/results.jsonl`'s
interpretation below ‚Äî retry is pending session-limit reset.
